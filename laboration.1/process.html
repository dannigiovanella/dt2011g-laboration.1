Laboration 1

Arbetsprocessen

Node.js och Vite

I det här arbetet har Node.js använts som en grund att kunna installera och använda olika utvecklingsverktyg för automatisering. Med hjälp av npm (Node Package Manager) har Vite installerats och använts för att optimera källkodsfiler genom att minifiera kod och optimera bilder innan webbplatsen publiceras.

1. Hantering av källkodsfiler

I arbetet med webbplatsen har strukturerats upp med källkodsfiler i HTML, CSS, JavaScript samt bilder. Dessa ligger separat från versionerna av dessa filer som Vite automatiskt skapar och lägger i en distrubitionsmapp vid optimering.

Under utvecklingen har Git används som versionshanterare kopplat mot Github, och här har det jobbats i främst två olika brancher, development som används under utvecklinen och main för att publicera färdiga och optimerade filer ifrån, dessa är de filer som Vite automatiskt skapar vid build-processen.

Under arbetet har regelbundna commits gjort för att dokumentera ändringar i arbetet. Vid installation av Vite skapades en gitignore-fil automatiskt där node-modules hamnar för att exkluderas i versionshanteringen för att undvika att skicka med onödiga filer.

2. Automatisering med Vite

För att underlätta arbetet med att optimera filer använde jag Vite som installerades med npm install vite --save-dev , och inställningar för hur arbetet skulle byggas med vite.config.js

Då vite har som standard att behandla projekt som en-sidorsprojekt så behövdes det göras en anpassning för Vite att hitta flera sidor, då arbetet innehåller tre sidor. Detta lösets med resolve som talar om vilka sidor och sökvägen till dessa som Vite ska behandla.

Plug-inet Vite image optimizer har använts för bilderna i arbetet med WebP-format, och har testats med olika kompressionsnivåer för att få bästa optimering kontra kvalité.

För att bygga det färdiga arbetet med inställningarna i Vite har npm run bild körts som skapar skapar filerna i en dist-mapp som är klara för publicering. Processen gör att JavaScript och CSS-filer slås ihop och minifierar koden, samt att bidlerna optimeras efter den nivå som är inställd i plugin för detta.

3. Publicering och test

Innan publicering testades webbplatsen genom att köra kommando npm run preview för att se hur webbplatsen såg ut efter processen med optimeringen.

I detta läge justerades även optimeringsnivån ytterligare på bilderna då dessa fortfarande höll så god kvalité att de kunde komprimeras ytterligare.

Efter justeringar när webbplatsen kändes klar gjordes en build-process på nytt och skapade den dist-mappen som var redo att publiceras. En merge gjored därefter med innehållet från development-branchen till huvudbranchen main. I detta läge pushades hela innehållet från main till Github för att kunna publiceras.

Färdiga filer i dist-mappen publicerades via Render som tillåter att ändingar som görs i main-branchen automatsikt uppdateras och syns på den publicerade webbplatsen.
